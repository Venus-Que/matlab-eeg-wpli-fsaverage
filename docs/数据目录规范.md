# 数据目录规范

## 推荐目录

批处理脚本从`data_root`递归查找：

```text
cohort/sub-001/phase1/1.2_ICA/ica_clean.fif
```

事件表默认位于同一阶段目录：

```text
cohort/sub-001/phase1/1.1_标签清洗/markers.csv
```

允许在`cohort`之前存在更高层目录，但从FIF向上解析时应依次得到：

```text
1.2_ICA -> phase -> subject -> cohort
```

## markers.csv

推荐4列格式：

```csv
index,time_seconds,event_id,label
1,10.500,1,O1
2,20.500,2,S1
```

也兼容3列格式：

```csv
time_seconds,event_id,label
10.500,1,O1
20.500,2,S1
```

代码按列名优先识别时间和标记列，并提供旧格式位置回退。时间单位必须为秒。

## 事件规则

- 含O/S事件时，对每个阶段分别建立`O1-O6`和`S1-S6`两条条件记录。
- O和S各自独立切试次、质量控制、频谱估计和连接计算，不合并原始试次。
- `markers.csv`中缺少某一条件时，该条件标记为`失败-缺少条件事件`，另一条件仍可继续。
- 如实验设计需要其他前缀，可修改批处理顶部的`condition_prefixes`；单条计算函数接受任意单个英文字母前缀。
- 对完全没有O/S但存在P事件的旧phase2记录，批处理只建立P条件记录，并将`condition_rule`标记为“仅P事件回退”。程序不会为不存在的O/S事件制造结果。

扫描表中的`marker_event_count`用于初步核对条件事件数，最终仍以单条计算时打印和成功切出的事件窗为准。

当前项目只读审计共发现130份事件表：98份同时含O和S，32份phase2仅含P。因而全量条件记录数不会简单等于130×2。

## 通道规则

本版本支持每条ICA FIF包含59个或64个EEG通道。59导记录会生成59×59矩阵和图片，但图标题及CSV会标注“仅个体展示”；其他导数会被标记为`跳过-不支持的导数`。

这里的导数只统计EEG通道。若文件含59个EEG加5个ECG/EOG辅助通道，脑网络仍是59节点，辅助通道不得作为脑网络节点，也不得补成64节点。

59导和64导具有不同节点集合，不能直接合并矩阵或逐边比较。正式组间分析应限定同一电极布局，或先定义所有记录共有的EEG节点集合并重新计算。
