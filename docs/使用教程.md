# 使用教程

## 1. 下载依赖

安装MATLAB并下载FieldTrip。只把FieldTrip根目录加入路径，不要使用`addpath(genpath(...))`：

```matlab
addpath('path/to/fieldtrip');
ft_defaults;
```

另行准备FreeSurfer fsaverage目录。

## 2. 检查环境

```matlab
repo_root = 'path/to/matlab-eeg-wpli-fsaverage';
addpath(fullfile(repo_root, 'tools'));
check_environment('path/to/fieldtrip', 'path/to/fsaverage');
```

## 3. 单条记录

打开`examples/run_single_subject_example.m`，填写目录和脱敏ID。运行后检查命令窗口：

- EEG通道数应为59或64；59导结果只作个体展示；
- 程序保留输入FIF中的实际EEG通道，不会自动把59导补成64导或把64导删成59导；
- 事件名称应符合当前阶段；
- 剔除后应至少有6个试次；
- 输出矩阵不是全NaN；
- 二维图和三维图不是空白。
- MAT中应同时存在`delta_matrix`、`theta_matrix`、`alpha_matrix`和`beta_matrix`；
- 二维汇总图应有8个面板，fsaverage图应有4个频段面板。

## 4. 批处理第一阶段：仅扫描

打开`batch/run_batch_wpli_fsaverage.m`并填写：

```matlab
fieldtrip_dir = 'path/to/fieldtrip';
data_root = 'path/to/preprocessed-data';
output_root = 'path/to/output';
fsaverage_dir = 'path/to/fsaverage';
```

保持：

```matlab
run_mode = 'all';
scan_only = true;
```

运行后检查扫描表中的组别、被试、阶段、事件前缀和文件路径。

## 5. 批处理第二阶段：预实验

修改：

```matlab
run_mode = 'pilot';
pilot_count = 4;
scan_only = false;
```

预实验不是只看程序是否报错，还应检查事件数量、保留试次数、矩阵分布、网络图和耗时。

## 6. 批处理第三阶段：全量

只有预实验通过后修改：

```matlab
run_mode = 'all';
scan_only = false;
overwrite_existing = false;
```

程序每完成一条就更新进度CSV。中断后重新运行时，已有完整结果会跳过。

### 从旧双频段结果升级

v0.1.x的MAT只含theta和alpha。新版批处理会读取MAT变量清单，只有同时存在delta、theta、alpha、beta四个矩阵，且节点映射记录了四个频段时，才判定为“已有完整结果”。因此保持`overwrite_existing=false`即可：旧双频段记录会重算，完整四频段记录会跳过，只有脑图不完整时会复用MAT补绘脑图。

## 7. 状态解释

| 状态 | 含义 |
|---|---|
| 成功 | 新完成计算和所需绘图 |
| 成功-59导仅个体展示 | 生成59×59矩阵和图片，标题已标注，不进入64导直接比较 |
| 跳过-已有结果 | MAT和所需图片已存在 |
| 跳过-已有59导结果 | 已有59导结果，仅用于个体展示 |
| 跳过-不支持的导数 | EEG通道数不是59或64，不进入本轮分析 |
| 失败 | 事件、试次数、坐标或其他质量门槛未通过 |
| 待运行 | 尚未处理 |

进度表的`band_count`列用于记录完整结果的频段数；当前新版成功结果应为4。

失败原因写入`message`列。不要为了减少失败数量而自动降低单个被试的质量标准。

## 8. 59导与64导的联合分析

项目数据包含19条64导记录和12条59导记录，两套固定布局只有54个同名公共通道。当前输出仍分别为64×64和59×59，不能直接求组均值或逐边比较。

需要联合比较时，应先确定54个公共通道，然后从原始FIF中统一选择这些通道并重新计算54×54矩阵。不要对现有59×59矩阵补5行5列，也不要对缺失边填0或插值。
